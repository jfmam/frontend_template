Bundler 학습을 두 부분으로 나누기

먼저 core를 직접 만들고, 그 다음 이력서에 쓸 수 있는 성능 개선 언어로 연결합니다.

Part 1. Bundler Core
1. 왜 bundler가 필요한가
브라우저 실행 모델, 모듈 의존성, 파일 타입 변환, 배포 산출물 문제
2. Mini Webpack
entry -> dependency graph -> transform -> bundle runtime -> rebuild
3. Mini Vite
dev server -> native ESM -> import rewrite -> on-demand transform -> HMR
Part 2. 선택 기준과 이력서 언어
1. Webpack vs Vite vs Rsbuild
번들링 시점, dev server 모델, rebuild 범위, transform 엔진 비교
2. 성능 실험
cold build, dev start, file change, HMR/reload 시간을 작은 예제로 측정
3. Resume Notes
"도구 교체"가 아니라 "병목 분석 -> 아키텍처 선택 -> 수치 개선"으로 정리
A

Core만 먼저 구현

가장 추천. mini-webpack과 mini-vite의 최소 동작을 만들고, 차이를 몸으로 이해합니다.

B

Core + 메모 문서 동시 진행

구현하면서 각 단계마다 이력서/면접 포인트를 바로 기록합니다.
