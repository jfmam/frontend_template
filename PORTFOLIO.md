## 소개

사용자 불편이 실제 매출 저하나 운영 실패로 이어지는 문제를 프론트엔드 관점에서 해결해왔습니다.  
이커머스 SaaS와 크롬 익스텐션 환경에서 대량 상품 업로드, 데이터 수집, 초기 로딩, 국제화 같은 병목을 개선하며 성능과 운영 안정성을 함께 높였습니다.

- 대량 옵션 편집 화면 최적화로 DOM 노드 수 84% 감소
- 코드 스플리팅으로 초기 번들 크기 60% 감소
- 카테고리 정책 데이터 로컬화로 외부 API 의존도를 낮추고 대량 업로드 안정성 확보
- Babel AST 기반 i18n 자동화 도구 개발로 국제화 적용 리드타임 85% 단축
- Chrome Offscreen 도입으로 상품 수집 시간 60% 단축
- Webpack에서 Rspack으로 전환하여 빌드 속도 90% 개선

## 핵심 역량

- 성능 최적화: 렌더링 병목, 번들 크기, DOM 노드 수, 빌드 시간 등 사용자 경험과 개발 생산성에 직접 영향을 주는 문제를 수치로 개선했습니다.
- 문제 해결: 단순 구현보다 병목 원인을 먼저 분석하고, 서비스 특성에 맞는 구조적 해결책을 선택했습니다.
- 운영 관점 개발: 상품 등록 실패, API Rate Limit, 수집 지연처럼 실제 운영 이슈를 추적하고 안정적인 대안을 만들었습니다.
- 자동화: 반복 작업을 줄이는 도구를 만들어 팀의 리드타임과 유지보수 비용을 함께 줄였습니다.

## 프로젝트 1. 퍼센티

### 서비스 소개

구매대행 셀러를 위한 올인원 SaaS 서비스입니다. 상품 수집부터 마켓 등록, 주문 관리, 애널리틱스, 상품 결제까지 셀러의 운영 전 과정을 지원합니다.

### 주요 업무

- 상품 업로드 플로우 전반 개발: 소싱, 편집, 업로드, 주문 관리
- 외부 마켓 API 연동을 통한 상품 결제 및 배송 추적 기능 개발
- 유저 불편 사항 추적 및 대응

### 주요 성과

#### 1. 상품 대량 옵션 편집 성능 개선

옵션 수가 많아질수록 상품 편집 화면이 멈춰 업로드 자체가 불가능한 문제가 있었습니다.

- 리스트 가상화를 도입해 뷰포트 외부 트리 렌더링 비용을 줄이고 React 재조정 단계의 부담을 낮췄습니다.
- DOM 노드 관리 구조를 최적화해 브라우저가 처리해야 하는 노드 양을 줄였습니다.
- 그 결과, 기존 대비 DOM 노드 수를 84% 줄여 대량 옵션 상품도 편집 가능한 수준으로 개선했습니다.

#### 2. 코드 스플리팅으로 초기 번들 크기 60% 감소

초기 `main.js`에 불필요한 리소스가 포함되어 사용하지 않는 자바스크립트 로드와 API 호출이 함께 발생하고 있었습니다.

- Bundle Analyzer로 초기 렌더링을 저해하는 대용량 의존성을 분석했습니다.
- 비즈니스 로직과 분리 가능한 대형 라이브러리인 `ckeditor`, `exceljs`와 일부 모달 컴포넌트에 지연 로딩을 적용했습니다.
- 그 결과, 초기 번들 기준 60%를 줄여 첫 진입 성능과 불필요한 리소스 로드를 개선했습니다.

#### 3. 카테고리 정책 데이터 로컬화로 대량 업로드 안정성 확보

네이버 커머스 API의 초당 호출 제한 때문에 여러 유저의 대량 업로드가 동시에 발생하면 `429` 에러와 등록 실패가 발생했습니다.

- 카테고리 정책 데이터는 변경 빈도가 낮고, 변경 시 사전 공지 기간이 있다는 운영 특성을 파악했습니다.
- 정책 데이터를 정적으로 관리하는 방향으로 전환하고, 정책 변경 시 재사용 가능한 추출 로직을 작성했습니다.
- 카테고리별 제한 정책 데이터를 크롤링해 정적 파일로 생성함으로써 외부 API 의존도를 줄였습니다.
- 그 결과, 대량 상품 업로드 과정의 안정성을 높이고 API 호출 제한으로 인한 실패 가능성을 낮췄습니다.

#### 4. Babel AST 기반 i18n 자동화 도구 개발

중국 시장 진출을 위한 국제화 과정에서 모든 텍스트를 수동으로 찾아 `t()` 함수로 감싸야 하는 반복 작업이 필요했습니다.

- Babel AST를 활용해 소스 코드를 파싱하고 텍스트 노드를 탐지하는 코드 변환 도구를 개발했습니다.
- 자동으로 `t()` 함수를 적용하는 Transform 파이프라인을 구성해 수작업을 크게 줄였습니다.
- 그 결과, 국제화 적용 리드타임을 약 85% 단축했습니다. `7일 -> 1일`

## 프로젝트 2. 크롬 익스텐션

### 서비스 소개

외부 서비스 데이터를 실시간으로 가공해 해외 상품 수집과 키워드 분석 등 셀러 업무에 최적화된 기능을 제공하는 크롬 익스텐션입니다.

### 주요 업무

- 해외 마켓 상품 스크래핑 기능 개발
- 유저에게 필요한 외부 데이터 수집 기능 개발

### 주요 성과

#### 1. Offscreen 도입으로 수집 시간 60% 단축

기존에는 데이터를 수집하기 위해 Chrome Tab을 생성하는 구조였고, 백그라운드 탭 상태에서는 CPU 점유율이 낮아져 수집 속도가 크게 저하됐습니다.

- Chrome Offscreen은 탭 상태와 관계없이 일정한 성능을 유지한다는 점을 확인했습니다.
- 데이터 처리 작업을 탭 기반 구조에서 Offscreen으로 이전했습니다.
- 그 결과, 상품 수집 시간을 60% 단축했습니다.

#### 2. Webpack에서 Rspack으로 전환하여 빌드 속도 90% 개선

익스텐션 테스트는 `빌드 -> 업로드 -> 새로고침` 흐름이 반복되기 때문에 빌드 시간이 곧 개발 속도와 직결되는 상황이었습니다.

- 자바스크립트 기반 번들러의 성능 한계를 해결하기 위해 Rust 기반의 Rspack 도입을 검토했습니다.
- 익스텐션 개발에 필요한 기존 플러그인 호환성을 확인한 뒤 빌드 도구를 전환했습니다.
- 그 결과, 빌드 속도를 90% 개선해 개발 피드백 루프를 크게 단축했습니다.
